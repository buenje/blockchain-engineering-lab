# 28-Day Solidity Challenge

Tägliche Coding-Übungen, um den Sprung von „Konzepte verstehen" zu „eigenständig vom leeren File aus coden" zu schaffen.

## Regeln

- **Mo–Sa, mindestens 1 Stunde Coding.** Sonntags frei.
- **Pattern aus dem Kopf reproduzieren**, nicht nur Videos konsumieren.
- **Pro Tag ein Ordner** (`day-01/`, `day-02/`, ...) mit Solidity-Datei und Reflexions-README.
- **Ein Commit pro Tag.** Auch wenn der Code grottig ist – gerade dann.
- **README ehrlich ausfüllen.** Vorlage in `_template/README.md`.

## Lernsequenz (sequenziell, nicht parallel)

1. Mappings mit Enum-Keys
2. Events
3. Modifier
4. Structs
5. Inheritance
6. (aktuell:) Selbstständiges Coding vom leeren File

## Verbindung zur Gesamt-Roadmap

Phase 1 (Monate 1–6): Cyfrin-Grundlagen, Foundry-Testing, eigenständig geschriebenes WorkflowPFV v2.

Diese Challenge ist Teil von Phase 1.
